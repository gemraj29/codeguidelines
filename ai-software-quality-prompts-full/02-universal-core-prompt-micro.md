# 02 – Universal Micro Prompt

Use clean architecture (UI/API → services → domain → infra).  
Keep business logic out of controllers/UI.  
Write readable, idiomatic, simple code.  
Consider time/space complexity before coding.  
Validate all inputs; enforce authorization.  
Do not log or hardcode secrets/PII.  
Add unit and integration tests.  
Explain the architecture briefly before code.
